Agent Loop：Spec → Coding → Verify

自主编程需要完整闭环，而非把问题抛给用户

01 Spec
02 Coding
03 Verify

Spec 阶段

澄清需求，明确验收标准，生成技术规格书，确保双方对"完成"的定义达成共识

Coding 阶段

根据 Spec 自主实现功能，无需用户持续监督，自主推进任务

Verify 阶段

自动运行测试验证实现，不符合 Spec 则自动迭代，而非把问题抛给用户

"AI 写代码、人来测试，不算自主编程。"
Agent Loop 06 / 10

自动适配复杂度，而非堆砌功能

动态加载 Skills

当任务涉及特定框架时，Quest 动态加载封装了工程实践的 Skills。

  • TypeScript 配置最佳实践
  • React 状态管理模式
  • 数据库索引常见陷阱
  • API 设计规范

团队也可将工程规范封装成 Skills：代码风格、Git 提交规范、测试覆盖率、安全审查清单

智能模型路由

当单一模型能力不足以覆盖任务需求时，Quest 自动调度多个模型协同工作。

有的模型擅长推理，有的擅长写作，有的擅长处理长上下文。智能路由根据子任务特性选择最合适的模型，对用户来说面对的始终是一个 Quest。

多 Agent 架构

主 Agent 负责规划协调，子 Agent 执行具体任务。保持克制使用——只在确实需要时启用。

项目理解内化

Quest 能深度分析项目的代码结构、架构演进、团队规范，将这些信息内化为"项目理解"。

  • 理解项目模块划分和依赖关系
  • 识别代码风格和命名习惯
  • 学习项目特定的架构模式
  • 掌握团队的工程实践

自我学习能力

面对陌生的 API 或新框架，Quest 通过探索和实践进行自我学习：

阅读文档 尝试调用 分析错误 调整方案

使用时间越长，它对项目理解越深，表现也越好。Quest 不仅执行任务，还会在执行中不断学习。
